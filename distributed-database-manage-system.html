Introduction to Distributed DBMS

A Distributed Database Management System (DDBMS) governs the storage and processing of logically related data over interconnected computer systems where both data and processing functions are distributed among several sites.

DDBMS evolved from centralized database systems to address the limitations of storing all corporate data in a single central site. The dynamic business environment created a demand for applications that could access data from different sources at multiple locations.

Key Insight: DDBMS allows for data to be located near the "greatest demand" site, enabling faster data access, improved communications, and reduced operating costs while maintaining data integrity.

DDBMS Architecture Diagram

Advantages of DDBMS

  • Data located near greatest demand site
  • Faster data access and processing
  • Growth facilitation - new sites can be added easily
  • Improved communications between departments
  • Reduced operating costs
  • Less danger of single-point failure

DDBMS Challenges

  • Complexity of management and control
  • Increased security concerns
  • Lack of standards across different systems
  • Increased storage requirements
  • Higher training costs
  • Distributed concurrency control issues

Data Fragmentation Strategies

Horizontal Fragmentation

Division of a relation into subsets (fragments) of tuples (rows). Example: Dividing CUSTOMER table by state.

Vertical Fragmentation

Division of a relation into attribute (column) subsets. Example: Splitting customer data from financial data.

Mixed Fragmentation

Combination of horizontal and vertical strategies. Example: Dividing customers by state and then splitting attributes.
